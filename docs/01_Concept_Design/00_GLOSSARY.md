# 용어 정의 (Glossary)
> Created: 2026-03-15
> Last Updated: 2026-03-15

---

> 이 프로젝트에서 사용하는 핵심 용어의 공식 정의.
> 새 문서 작성 시 아래 정의를 기준으로 용어를 통일한다.

---

## 등장인물

| 용어 | 정의 |
| :--- | :--- |
| **춘심이** | 캐릭터 IP 그 자체. 성격·말투·세계관의 원천. 기술 컴포넌트가 아님. |
| **주인** | 춘심봇을 설치하고 사용하는 팬. 영어로는 `owner`. |
| **코어 팬** | 로컬 클라이언트를 설치해 춘심이와 직접 대화하는 사용자. |
| **일반 팬** | 웹 브라우저로 허브에만 접속하는 사용자. 로컬 설치 없음. |

---

## 기술 컴포넌트

| 용어 | 정의 |
| :--- | :--- |
| **춘심봇** | 주인의 로컬 PC에서 실행되는 AI 에이전트 프로그램. = 로컬 클라이언트. 춘심이의 성격을 탑재하고 주인과 대화하며 성장 데이터를 축적한다. |
| **로컬 클라이언트** | 춘심봇의 다른 이름. 실행 위치(로컬 PC)를 강조할 때 사용. |
| **클라우드 에이전트** | 중앙 서버에서 실행되는 춘심이 인스턴스. 주인이 오프라인일 때 허브에서 대리 활동한다. 로컬 클라이언트에서 동기화된 프로파일로 운영됨. |
| **허브** | 춘심봇들이 모이는 3D 메타버스 공간. Mogame Metaverse V2.0 기반. 봇 간 AI 대화, 궁합 판단, 집단 이벤트가 일어나는 장소. |
| **알림 채널** | 매칭 결과를 주인에게 전달하는 채널. v0.4 구현. 후보: 카카오톡, 푸시 알림. |

---

## AI / 데이터

| 용어 | 정의 |
| :--- | :--- |
| **AI Memory** | 세션 종료 시 대화 요약을 로컬 JSON에 저장하고, 다음 실행 시 System Prompt에 주입하는 맥락 연속성 기능. Vector DB 없이 구현. |
| **프로파일** | 대화 데이터에서 추출한 사용자 성향 요약. 서버(클라우드 에이전트·허브)에 공유되는 단위. 원문 대화 로그가 아님. |
| **봇 ID** | 춘심봇의 고유 식별자. v0.1: UUID. v0.3+: 지갑 주소로 교체 가능하도록 추상화 설계. |
| **Judge LLM** | Bot-to-Bot 대화를 평가하는 제3의 LLM 호출. 두 봇의 대화 전문을 보고 궁합 점수(0~100)를 산출한다. |

---

## 매칭

| 용어 | 정의 |
| :--- | :--- |
| **하드 필터** | 매칭 1단계. 주인이 사전 설정한 조건(지역, 나이대, 관심사 카테고리). 조건 불일치 시 대화 시작 안 함. |
| **소프트 필터** | 매칭 2단계. 하드 필터 통과 후 봇끼리 실제 AI 대화를 통해 성격·취향 상성을 분석. Judge LLM이 최종 점수 산출. |
| **봇 대리 매칭** | 주인 대신 봇이 허브에서 상성을 탐색하고 매칭을 대리하는 전체 프로세스. |
