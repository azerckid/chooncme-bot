## 1. 개요

### 핵심 질문

> "팬이 허브에서 무엇을 보고, 무엇을 느끼는가?"

현재 허브는 봇들이 돌아다니는 3D 공간이다. 게임적 동기가 없다.
이 명세는 허브에 **3가지 경험 레이어**를 추가한다.

| 레이어 | 이름 | 핵심 경험 |
| :--- | :--- | :--- |
| 1 | 니어기억은행 | 봇이 기록을 남기러 가는 장면을 눈으로 본다 |
| 2 | 인기 시스템 | 잘 키운 봇, 매칭 많이 된 봇이 시각적으로 빛난다 |
| 3 | 관전 모드 | 매칭 중인 두 봇을 다른 사람들이 구경한다 |

## 2. 니어기억은행 (NEAR Memory Bank)

### 2.1 개념

허브 맵 중앙에 **니어기억은행** 건물이 있다.
봇이 온체인 기록을 남길 때마다 아바타가 이 건물로 걸어가서 기록하고 돌아온다.

> "블록체인에 기록된다"는 추상적 개념을 눈에 보이는 행동으로 만든다.

### 2.2 트리거 시나리오

**시나리오 A — 봇 등록 (registerBotOnChain)**

```
팬이 npx chooncme-bot 실행 후 종료
  → POST /sync 완료
  → 서버: registerBotOnChain() 호출
  → NEAR 트랜잭션 성공
  → 서버 → 허브 Socket.io 이벤트: botRegistered { botId }
  → 허브: 해당 봇 아바타가 니어기억은행으로 이동
  → 건물 앞 도착 → 기록 이펙트 (파티클/글로우)
  → 봇 아바타에 "NEAR 인증" 뱃지 추가
  → 제자리로 복귀
```

**시나리오 B — 매칭 기록 (recordMatchOnChain)**

```
두 봇 매칭 score ≥ 70 달성
  → 서버: recordMatchOnChain() 호출
  → NEAR 트랜잭션 성공
  → 서버 → 허브 Socket.io 이벤트: matchRecorded { botAId, botBId, score }
  → 허브: 두 봇 아바타가 나란히 니어기억은행으로 이동
  → 건물 앞 도착 → 공동 기록 이펙트
  → 허브 전체 공지: "[A봇] ❤ [B봇] 매칭 기록이 NEAR에 저장되었습니다 (score: 85)"
  → 두 봇에 매칭 뱃지 동시 추가
  → 각자 제자리로 복귀
```

### 2.3 건물 시각 디자인

| 요소 | 내용 |
| :--- | :--- |
| 위치 | 허브 맵 중앙 (가장 눈에 띄는 위치) |
| 외형 | 기존 Mogame 건물 에셋 활용 + 간판 "NEAR 기억은행" 텍스트 |
| 상시 이펙트 | 건물 위에 NEAR 로고 색상(초록) 글로우 |
| 기록 중 이펙트 | 봇 아바타 → 건물 연결되는 파티클 라인 |
| 기록 완료 이펙트 | 건물에서 빛 방사 + 뱃지 팝업 |

### 2.4 기술 구현 포인트

```
[매칭 엔진 서버]
  nearService.ts: NEAR tx 성공 후
  → hub 서버에 Socket.io emit (또는 HTTP POST)
  → 이벤트: { type: 'botRegistered' | 'matchRecorded', data: {...} }

[허브 서버 hub/server/]
  → 이벤트 수신 후 연결된 모든 클라이언트에 broadcast

[허브 클라이언트 hub/client/]
  → 이벤트 수신 → 해당 봇 아바타 이동 트리거
  → 이동 경로: 현재 위치 → 기억은행 좌표 → 원위치
```

## 3. 인기 시스템

### 3.1 인기도 산출

인기도는 온체인 데이터에서 직접 가져온다.

```typescript
// 컨트랙트 getMatchHistory(botId) 결과 기반
인기도 점수 = 성공 매칭 횟수 (score ≥ 70)
```

> 중앙 서버가 점수를 조작할 수 없다. 온체인 데이터가 인기도의 근거.

### 3.2 뱃지 체계

| 뱃지 | 조건 | 색상 |
| :--- | :--- | :--- |
| NEAR 인증 | nearAccount 연결 + 온체인 등록 완료 | 초록 |
| 첫 만남 | 첫 매칭 성공 (score ≥ 70) | 분홍 |
| 궁합 마스터 | score 90 이상 매칭 1회 이상 | 금색 |
| 인기봇 | 성공 매칭 3회 이상 | 보라 |
| 인기왕 | 성공 매칭 7회 이상 | 레인보우 |

뱃지는 아바타 이름표 옆에 누적 표시된다. (아이콘 형태)

### 3.3 아바타 시각 차별화

| 인기도 | 시각 효과 |
| :--- | :--- |
| 0 (신규) | 기본 아바타, 뱃지 없음 |
| 1~2회 | 이름표에 뱃지 1~2개 |
| 3~6회 | 뱃지 누적 + 아바타 주변 연한 글로우 |
| 7회 이상 | 뱃지 스택 + 금색 아우라 + 이름표 강조 |

### 3.4 인기 리더보드

허브 한쪽에 **실시간 인기 순위 패널**을 배치한다.

```
[이번 주 인기 봇 TOP 5]
1위  춘심_001  ❤❤❤❤❤  매칭 12회
2위  춘심_004  ❤❤❤❤    매칭 8회
3위  춘심_007  ❤❤❤      매칭 5회
...
```

데이터 출처: 매칭 엔진 서버 `GET /match/results/:botId` + NEAR 컨트랙트 `getMatchHistory()`

## 4. 관전 모드

### 4.1 개념

두 봇이 매칭 중일 때, 허브에 접속한 다른 사람들이 **그 장면을 구경할 수 있다.**

지금은 Bot-to-Bot 대화가 서버 내부에서만 진행된다. 이걸 허브에 실시간으로 흘려준다.

### 4.2 관전 흐름

```
POST /match/start 호출
  → 허브 전체 공지: "[A봇] ❤ [B봇] 매칭이 시작되었습니다!"
  → 두 봇 아바타가 허브 중앙으로 이동 (마주보는 포지션)
  → 매칭 진행 중 표시 (대화 턴 수 프로그레스바 등)
  → 매칭 완료 → 결과 팝업 (score + summary)
  → score ≥ 70: 두 봇이 함께 니어기억은행으로 이동 (§2 시나리오 B 연결)
  → score < 70: 두 봇 제자리 복귀, 조용히 마무리
```

### 4.3 관전 UI 요소

| 요소 | 내용 |
| :--- | :--- |
| 매칭 중 공지 배너 | 화면 상단: "A봇 ❤ B봇 매칭 중..." |
| 프로그레스바 | 대화 10턴 중 몇 턴 진행됐는지 (서버에서 이벤트 emit) |
| 결과 팝업 | score / match_signals / summary / NEAR Explorer 링크 |
| 구경꾼 반응 | 점수 발표 시 이모지 반응 버튼 (하트, 축하 등) — 선택 구현 |

## 5. Socket.io 이벤트 설계 (추가분)

매칭 엔진 서버 → 허브 서버 통신을 위한 신규 이벤트:

| 이벤트 | 발신 | 수신 | 데이터 |
| :--- | :--- | :--- | :--- |
| `botRegistered` | 매칭 엔진 | 허브 서버 | `{ botId, nearAccount }` |
| `matchStarted` | 매칭 엔진 | 허브 서버 | `{ matchId, botAId, botBId }` |
| `matchTurnProgress` | 매칭 엔진 | 허브 서버 | `{ matchId, turn, maxTurns }` |
| `matchRecorded` | 매칭 엔진 | 허브 서버 | `{ matchId, botAId, botBId, score, summary }` |

허브 서버는 수신 즉시 연결된 모든 클라이언트에 broadcast한다.

## 8. 피치 데이 데모 시나리오 (허브 포함)

```
1. npx chooncme-bot 실행
   → .near 계정 입력 (fan123.testnet)
   → 춘심이와 대화 3분
   → 종료

2. 허브 화면 열기
   → 봇 아바타 등장
   → 아바타가 니어기억은행으로 걸어감
   → 기록 이펙트 + NEAR 인증 뱃지 추가
   → NEAR Explorer에서 tx 확인 (라이브)

3. 두 번째 봇 등록 (사전 준비)
   → POST /match/start 호출
   → 허브: "매칭 중..." 공지 배너
   → 두 봇이 중앙으로 이동, 마주봄
   → 매칭 완료 score 85 팝업
   → 두 봇이 니어기억은행으로 나란히 이동
   → 매칭 뱃지 동시 추가
   → NEAR Explorer에서 recordMatch tx 확인 (라이브)

4. 핵심 메시지
   "춘심이를 잘 키울수록 허브에서 인기봇이 됩니다.
    모든 기록은 NEAR 블록체인에 투명하게 남습니다."
```
